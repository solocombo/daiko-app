# 代購 DAIKO 管理系統 — 部署教學

## 總覽
本系統使用：
- **React + Vite**：前端框架
- **Supabase**：雲端資料庫（兩人同步）
- **GitHub**：存放程式碼
- **Vercel**：部署網頁（產生可用的網址）

全部免費 🎉

---

## 第一步：建立 Supabase 資料庫

1. 前往 [https://supabase.com](https://supabase.com) 註冊帳號（用 Google 即可）
2. 點「New Project」建立新專案
   - 名稱：`daiko-app`（隨意）
   - 密碼：設一個資料庫密碼（可以亂打，記下來備用）
   - 地區：選 **Northeast Asia (Tokyo)** 最近
3. 等待約 1 分鐘建立完成
4. 進入專案後，點左側選單的 **SQL Editor**
5. 貼上 `supabase-schema.sql` 的全部內容，點 **Run**
6. 看到 Success 表示資料表建立完成

### 取得連線金鑰
1. 點左側 **Project Settings** > **API**
2. 複製以下兩個值，等一下要用：
   - **Project URL**（例：`https://abcdef.supabase.co`）
   - **anon public key**（很長的一串字）

---

## 第二步：上傳程式碼到 GitHub

1. 前往 [https://github.com](https://github.com) 註冊帳號（一個人做就好）
2. 點右上角「+」>「New repository」
3. 名稱：`daiko-app`，選 **Private**（私人，外人看不到）
4. 建立完成後，照畫面上的指示把程式碼上傳

   如果你有安裝 Git，在 `daiko-app` 資料夾執行：
   ```bash
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/你的帳號/daiko-app.git
   git push -u origin main
   ```

   如果沒有 Git，可以直接在 GitHub 網頁上傳資料夾（拖曳上傳）。

---

## 第三步：部署到 Vercel

1. 前往 [https://vercel.com](https://vercel.com) 用 GitHub 帳號登入
2. 點「Add New Project」
3. 選擇你剛才的 `daiko-app` repository，點「Import」
4. **重要：設定環境變數**
   - 找到「Environment Variables」區塊
   - 新增第一個：
     - Name：`VITE_SUPABASE_URL`
     - Value：貼上剛才的 Project URL
   - 新增第二個：
     - Name：`VITE_SUPABASE_ANON_KEY`
     - Value：貼上剛才的 anon public key
5. 點「Deploy」
6. 等待約 1 分鐘，Vercel 會給你一個網址（例：`https://daiko-app-xxx.vercel.app`）

### 兩個人都能用
- 把這個網址傳給你的合作夥伴
- 兩個人打開同樣的網址就能共用同一個資料庫
- 任何一方修改，另一方重新整理頁面就能看到

---

## 第四步：之後更新程式碼

每次修改程式碼後，只要推到 GitHub，Vercel 會自動重新部署，不需要手動操作。

---

## 常見問題

**Q：資料安全嗎？**
A：Supabase 的資料存在雲端，只有知道網址的人才能使用。如果要加上登入保護，可以之後再加。

**Q：資料會不見嗎？**
A：不會，Supabase 免費版有 500MB 空間，代購資料很小，用幾年都沒問題。

**Q：兩個人同時修改同一筆訂單會衝突嗎？**
A：最後儲存的人的版本會覆蓋，使用時稍微注意一下就好。
